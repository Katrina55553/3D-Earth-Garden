# 🐛 项目问题报告

## 🚨 严重问题

### 1. ESLint 配置错误
**位置**: `eslint.config.mjs`  
**问题描述**: 尝试导入 `eslint-config-next/core-web-vitals`，但实际文件是 `core-web-vitals.js`  
**影响**: 构建时出现警告  
**修复方案**:
```javascript
// 修改前
import nextVitals from "eslint-config-next/core-web-vitals";

// 修改后
import nextVitals from "eslint-config-next/core-web-vitals.js";
```

### 2. 模型文件体积过大
**位置**: `public/models/` 目录  
**问题描述**: 
- 7个 GLB 模型总大小 4.2MB
- 单个模型最大 777KB（妗夋爲.glb）
- 最小 424KB（鍖楃編绾㈡潐.glb）  
**影响**:
- 首次加载时间过长
- 带宽消耗大
- 移动端体验差  
**解决方案**:
- [ ] 使用 Draco 压缩模型
- [ ] 优化模型几何体和纹理
- [ ] 实现模型懒加载机制
- [ ] 考虑使用 CDN 加速

### 3. 文件命名问题
**位置**: `public/models/` 下的模型文件  
**问题描述**: 模型文件使用中文命名（如 `妗傝姳.glb`）  
**影响**:
- 跨平台兼容性问题
- 部署可能出现乱码
- 不利于版本管理  
**解决方案**:
```bash
# 重命名建议
妗傝姳.glb -> osmanthus.glb
妗夋爲.glb -> eucalyptus.glb
姣涚.glb -> bamboo.glb
楂樺北闆幉.glb -> snow_lotus.glb
鍖楃編绾㈡潐.glb -> redwood.glb
鐚撮潰鍖呮爲.glb -> baobab.glb
鐧惧悎.glb -> lily.glb
```

## ⚠️ 性能问题

### 4. 潜在内存泄漏风险
**位置**: `src/components/PlantModel.tsx` 第107-121行  
**问题描述**: 在 `useFrame` 中每帧都遍历整个 clonedScene  
**代码**:
```typescript
clonedScene.traverse((child) => {
  if (child instanceof Mesh && child.material) {
    // 频繁遍历所有子对象
  }
});
```
**影响**: 性能随模型复杂度指数级增长  
**解决方案**:
- [ ] 缓存遍历结果
- [ ] 使用 React.memo 优化组件
- [ ] 考虑使用 Three.js instancing
- [ ] 实现对象池管理

### 5. 重复的几何体创建
**位置**: `src/components/PlantModel.tsx` 第35行  
**问题描述**: 每个植物都创建相同的 RingGeometry  
**代码**:
```typescript
// 共享的草状地面光晕圆盘
const glowGeo = new RingGeometry(0.04, 0.14, 32);
```
**解决方案**: 移到组件外部或使用 useMemo 共享

### 6. 过多的植物实例渲染
**问题描述**: 51株植物同时渲染在地球上  
**影响**: 
- GPU 负载重
- 帧率不稳定
- 移动端卡顿  
**解决方案**:
- [ ] 实现视锥体剔除 (Frustum Culling)
- [ ] 使用 LOD (Level of Detail) 系统
- [ ] 距离较远的植物使用简化模型
- [ ] 根据屏幕大小动态调整渲染数量
