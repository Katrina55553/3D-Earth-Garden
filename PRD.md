# 项目 PRD：地球花园 3D 交互应用
## 项目概述
本项目是一个基于 Web 端的 3D 数据可视化应用。核心体验是通过一个可自由旋转、缩放的 3D 地球，展示世界各地的代表性植物 3D 模型。用户可以通过多维度的 UI 面板进行数据筛选，并在沉浸式的交互中查看植物详情。
技术栈要求：
- 前端框架：Next.js 加 React
- 3D 引擎：Three.js 加 React Three Fiber
- 样式方案：Tailwind CSS
- 动画库：Framer Motion 加 GSAP

## 核心里程碑拆解
### Phase 1: 基础环境搭建与数字地球点亮
- 目标：初始化项目，建立 3D 场景，让地球转起来。
- 具体任务：
1. 创建 Next.js 项目并配置 Tailwind CSS。
2. 安装 R3F 相关依赖。
3. 搭建全屏的 Canvas 画布。
4. 添加深空纯色背景或基础的星空粒子。
5. 渲染一个基础的 3D 球体代表地球，并添加 OrbitControls 以支持鼠标的拖拽旋转和滚轮缩放。
### Phase 2: 数据结构定义与模型种植
- 目标：建立植物数据结构，加载指定的本地模型，解决地理坐标到 3D 坐标的映射。
- 具体任务：
1. 定义植物数据的 JSON 结构，包含字段：ID、名称、经度、纬度、大洲、植被类型、气候特征、模型相对路径。
数据必须严格对应以下已准备好的 18 个本地模型文件：
桂花.glb、百合.glb、高山雪莲.glb、毛竹.glb、无花果.glb、金盏菊.glb、油橄榄.glb、高山火绒草.glb、猴面包树.glb、金合欢.glb、鹤望兰.glb、箭袋树.glb、巨人仙人掌.glb、龙舌兰.glb、北美红杉.glb、亚马逊王莲.glb、桉树.glb、帝王花.glb。
2. 编写核心数学函数：将现实世界中的经纬度转换为 3D 球体表面的笛卡尔坐标。
3. 利用 useGLTF 钩子加载上述的 .glb 植物模型文件。
4. 遍历 JSON 数据，将加载好的模型精准放置在地球表面的对应坐标上，并确保模型的法线垂直于地表，即植物向外生长。
### Phase 3: 镜头运镜与基础 3D 交互
- 目标：让场景具有生命力，实现鼠标与 3D 模型的互动。
- 具体任务：
1. Hover 态：鼠标悬停在某个植物模型上时，该模型稍微放大或带有外发光轮廓，鼠标指针变为手型。
2. Click 态：用户点击某个植物时，触发相机的平滑动画。镜头从当前位置平滑飞行并拉近对焦到被点击的植物位置。
### Phase 4: UI 覆盖层与数据联动控制台
- 目标：构建具有现代感的控制界面，并实现数据筛选逻辑。
- 具体任务：
1. 在 3D 画布上方叠加 HTML 与 UI 层。
2. 顶部统计区：显示标题及当前视角的植物数量统计。
3. 信息浮窗：点击植物后，在 UI 层弹出对应植物的详细信息卡片，包含名称和习性等，卡片支持关闭。
4. 底部控制台构建：使用 Tailwind 编写带有毛玻璃效果的控制台。包含大洲切换 Tab 和气候或类型下拉筛选器。
5. 数据联动逻辑：当用户在底部进行筛选时，3D 场景中不符合条件的植物模型平滑淡出变透明或变暗，符合条件的保持高亮。
